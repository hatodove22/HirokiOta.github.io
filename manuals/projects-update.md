## ⚠️ AI向け重大な注意書き（必須確認事項）

**この注意書きは、プロジェクト更新作業を行う際に、必ず最初に確認してください。**

**AIは以下のルールを厳守してください：**

### 更新作業開始前の必須確認

1. **必要な情報がすべて揃うまで更新作業を開始しない**
   - 以下の「ユーザーが用意すべき情報リスト」に記載されている情報が**すべて揃うまで**、更新作業を開始してはいけません。
   - 情報が不足している場合は、必ずユーザーに確認を取ってください。

2. **更新作業開始前の確認手順**
   - 更新作業を開始する前に、必ず以下の手順を実行してください：
     1. 以下の「ユーザーが用意すべき情報リスト」を確認
     2. ユーザーが提供した情報と照らし合わせて、不足している情報がないか確認
     3. 不足している情報がある場合は、**更新作業を開始せず**、ユーザーに不足している情報を明確に伝えて確認を取る
     4. **すべての必要な情報が揃ったことを確認できた場合のみ**、更新作業を開始する

3. **確認を取る際の注意点**
   - 不足している情報を箇条書きで明確に提示してください
   - ユーザーが「とりあえず進めて」などと言っても、必要な情報が不足している場合は、更新作業を開始しないでください
   - 情報が不足している場合は、「以下の情報が不足しているため、更新作業を開始できません。不足している情報を提供いただけますか？」という形で確認を取ってください

4. **情報が不足している場合の対応**
   - 情報が不足している場合は、**決して推測や仮の値で更新作業を進めないでください**
   - 情報が不足している場合は、**ファイルの作成・更新を行わないでください**
   - 情報が不足している場合は、**ユーザーに確認を取るまで待機してください**

## 用意しておく情報リスト

このポートフォリオは基本的にAIが更新することを前提としています。以下のリストは、ユーザーが用意すべき情報と、AIが適切に設定すべき情報を分けて記載しています。


### ユーザーが用意すべき情報リスト

プロジェクトを追加・更新する際に、ユーザーが提供すべき情報です。

#### 基本情報

- [ ] **プロジェクト名**（日本語・英語）
  - 例: 日本語「FresnelShape」、英語「FresnelShape」
- [ ] **公開日**（`YYYY-MM-DD` 形式）
  - 例: `"2025-03-01"`
- [ ] **タグ**（配列形式）
  - 例: `["Shape Display", "Haptics"]`
- [ ] **概要**（日本語・英語、短い説明文）
  - 例: 日本語「指先への触覚提示により、様々な形状情報を提示できる小型軽量なデバイス」
  - 例: 英語「A small and lightweight device that can provide various shape information through tactile feedback to fingertips.」

#### データソース（本文生成用）

- [ ] **PDFファイル**（論文、スライド、ポスターなど）
  - プロジェクトに関する情報が記載されたPDFファイル
- [ ] **ウェブサイトURL**（プロジェクトページ、デモサイトなど）
  - プロジェクトに関する情報が記載されたウェブページのURL
- [ ] **その他の資料**（テキストファイル、画像など）
  - プロジェクトに関する情報が記載されたその他の資料

**注意**: 本文内容はAIがデータソースから情報を抽出して自動生成します。ユーザーはデータソースを提供するだけで構いません。

#### 画像・アセット

- [ ] **メインビジュアル画像**（HeroImage）
  - 縦横比: 3:2（推奨）
  - 解像度: 1920px × 1280px（推奨）
  - ファイルサイズ: 1MB未満
  - 形式: PNG または JPEG
- [ ] **本文用画像**（GIF、説明図など）
  - 縦横比: 16:9（推奨）または用途に応じて自由
  - 解像度: 1600px × 900px（推奨）
  - ファイルサイズ: 2MB未満
  - 形式: PNG、JPEG、または GIF

#### リンク情報

- [ ] **リポジトリURL**（GitHubなど、オプション）
  - 例: `"https://github.com/example/project"`
- [ ] **デモURL**（オプション）
  - 例: `"https://demo.example.com"`
- [ ] **スライドURL**（オプション）
  - 例: `"https://slides.example.com"`

#### 関連情報

- [ ] **関連論文ID**（論文データに存在するIDの配列、該当がない場合は空配列）
  - 例: `["paper7", "paper12"]` または `[]`

**注意**: 
- **言語設定**: プロジェクトは必ず日本語と英語の2つのバージョンを作成する運用のため、言語設定は不要です。AIが自動的に `"both"` を設定します。
- **ピン留め設定**: 最近のプロジェクトが自動で追加されるため、ピン留め設定は不要です。AIが自動的に最新プロジェクトをトップページに表示するように設定します。

### AIが適切に設定すべき情報リスト

以下の情報は、AIがユーザーが提供した情報を基に自動的に生成・設定します。

#### 識別子・パス情報

- [ ] **ID**（一意のプロジェクトID）
  - AIが既存のIDを確認し、重複しない一意のIDを生成
  - 例: `"1"`, `"2"`, `"3"` など
- [ ] **スラッグ**（URL用の英数字）
  - AIがプロジェクト名から適切なスラッグを生成
  - 既存のスラッグと重複しないことを確認
  - 例: `"fresnelshape"`, `"hapnroll"` など
- [ ] **ディレクトリ名**（プロジェクトフォルダ名）
  - AIが `project-{id}-{slug}` の形式でディレクトリ名を生成
  - 例: `project-1-fresnelshape`
- [ ] **ファイルパス**（画像パスなど）
  - AIがディレクトリ構造に基づいて適切なパスを生成
  - 例: `"/content/projects/project-1-fresnelshape/images/HeroImage.png"`

#### 本文内容の生成

AIがユーザーが提供したデータソース（PDF、ウェブサイトなど）から情報を抽出し、以下の形式で本文を生成します：

- [ ] **背景**（`## 背景` / `## Background`）
  - プロジェクトの背景や動機を説明
- [ ] **アプローチ**（`## アプローチ` / `## Approach`）
  - どのようなアプローチを取ったかを説明
- [ ] **何がうれしいか**（`## 何がうれしいか` / `## Benefits`）
  - プロジェクトの成果や特徴、ユーザーにとってのメリットを説明
- [ ] **今後の展望**（`## 今後の展望` / `## Outlook`）
  - 今後の展開や展望を説明（オプション）

**生成ファイル**:
- `content/description_ja.md`（日本語版）
- `content/description_en.md`（英語版）

#### AIによる本文生成時の注意事項

AIは以下の注意事項に従って本文を生成してください：

##### 1. フォーマットの遵守

- 必ず以下の4つのセクションで構成してください：
  1. **背景**（`## 背景` / `## Background`）
  2. **アプローチ**（`## アプローチ` / `## Approach`）
  3. **何がうれしいか**（`## 何がうれしいか` / `## Benefits`）
  4. **今後の展望**（`## 今後の展望` / `## Outlook`）
- 各セクションは見出し（`##`）で始め、その下に本文を記載してください
- 画像の追加が必要な場合は、適切な位置にMarkdown形式で画像を挿入してください

##### 2. 読者層の想定

- **まったく専門外の人間が読むことを想定**してください
- 専門用語はできるだけ使用を避けてください
- 専門用語を使用する場合は、**用語のすぐ後にかっこ書きで簡潔な説明文を記載**してください
  - 例: 「ハプティクス（触覚提示技術）」
  - 例: 「VR（バーチャルリアリティ）」
  - 例: 「モータ（モーター）」

##### 3. 文体とスタイル

- **ですます調**で記述してください
- **わかりやすさを重視**してください
  - 長い文章は適度に分割してください
  - 複雑な概念は簡潔に説明してください
  - 具体例を交えて説明してください

##### 4. 既存プロジェクトの参照

- **既存のプロジェクトの文章を参照**してください
  - `content/projects/project-1-fresnelshape/content/description_ja.md` や `description_en.md` などを参考にしてください
  - 既存プロジェクトと同じトーンや文体を意識してください
  - 既存プロジェクトで使用されている専門用語の説明方法を参考にしてください

##### 5. データソースからの情報抽出

- PDF、ウェブサイトなどのデータソースから情報を抽出する際は、以下の点に注意してください：
  - 技術的な詳細は、わかりやすい言葉に置き換えてください
  - 専門的な概念は、一般の人にも理解できるように説明してください
  - データソースに記載されている情報をそのまま転記するのではなく、読者層に合わせて再構成してください

##### 6. 日英両方の生成

- 日本語版（`description_ja.md`）と英語版（`description_en.md`）の両方を生成してください
- 英語版は、日本語版を単純に翻訳するのではなく、英語圏の読者にもわかりやすいように再構成してください
- 専門用語の説明は、それぞれの言語で適切に表現してください

##### 7. 画像の追加

- データソースに画像やGIFが含まれている場合、適切な位置にMarkdown形式で画像を挿入してください
- 画像パスは、プロジェクトのディレクトリ構造に基づいて正しく設定してください
  - 例: `![Background](/content/projects/project-1-fresnelshape/images/background.gif)`

#### 自動設定項目

- [ ] **`status`**（公開状態）
  - デフォルト: `"Published"`
  - 下書きの場合は `"Draft"` を設定
- [ ] **`language`**（言語設定）
  - プロジェクトは必ず日本語と英語の2つのバージョンを作成する運用のため、常に `"both"` を設定
  - ユーザーからの指定は不要
- [ ] **`isPinned`**（ピン留め設定）
  - 最近のプロジェクトが自動でトップページに追加されるため、最新プロジェクトとして自動設定
  - ユーザーからの指定は不要
- [ ] **ディレクトリ構造の作成**
  - AIが以下のディレクトリ構造を自動生成：
    ```
    project-{id}-{slug}/
    ├── info/
    │   └── metadata.json
    ├── content/
    │   ├── description_ja.md
    │   └── description_en.md
    ├── images/
    └── assets/
    ```

#### AIの命名規則

AIは以下の規則に従って識別子を生成します：

1. **IDの生成**
   - 既存のプロジェクトIDを確認
   - 最大のIDに1を加えた値を新しいIDとして設定
   - 例: 既存の最大IDが `"5"` の場合、新規IDは `"6"`

2. **スラッグの生成**
   - プロジェクト名（英語）から生成
   - 小文字に変換
   - スペースや特殊文字をハイフンに変換
   - 既存のスラッグと重複しないことを確認
   - 例: "Fresnel Shape" → `"fresnel-shape"`

3. **ディレクトリ名の生成**
   - `project-{id}-{slug}` の形式
   - 例: ID `"1"`, スラッグ `"fresnelshape"` → `project-1-fresnelshape`

## 概要

プロジェクトは `content/projects/` ディレクトリ配下のファイルで管理されています。各プロジェクトは以下のディレクトリ構造を持ちます：

```
content/projects/
└── project-{id}-{slug}/
    ├── info/
    │   └── metadata.json       # メタデータ（タイトル、日付、タグなど）
    ├── content/
    │   ├── description_ja.md   # 日本語の本文
    │   └── description_en.md   # 英語の本文
    ├── images/                  # 画像ファイル（HeroImage、GIFなど）
    └── assets/                  # その他のアセットファイル
```

## 更新手順

### 1. メタデータの更新
AIが `info/metadata.json` を作成・編集して、プロジェクトの基本情報（タイトル、日付、タグなど）を更新します。

### 2. 本文の生成
AIがユーザーが提供したデータソース（PDF、ウェブサイトなど）から情報を抽出し、`content/description_ja.md` および `content/description_en.md` を生成します。

### 3. 画像・アセットの追加・更新
AIが `images/` または `assets/` ディレクトリに新しいファイルを追加、または既存ファイルを置き換えます。

### 4. 表示確認
開発サーバーで変更内容を確認し、問題がなければビルドして最終確認します。

## 各手順の詳細

### 1. メタデータの更新

#### ファイルの場所
```
content/projects/project-{id}-{slug}/info/metadata.json
```

#### 更新手順

1. **該当プロジェクトのディレクトリを開く**
   - 例: `content/projects/project-1-fresnelshape/info/metadata.json`

2. **`metadata.json` を開いて編集**

3. **各項目を更新**

#### メタデータの各項目の説明

| 項目 | 説明 | 必須 | 例 |
|------|------|------|-----|
| `id` | プロジェクトの一意のID | ✅ | `"1"` |
| `title` | プロジェクト名（日英対応） | ✅ | `{"ja": "FresnelShape", "en": "FresnelShape"}` |
| `slug` | URL用のスラッグ（英数字とハイフンのみ） | ✅ | `"fresnelshape"` |
| `status` | 公開状態 | ✅ | `"Published"` または `"Draft"` |
| `date` | 公開日（YYYY-MM-DD形式） | ✅ | `"2025-03-01"` |
| `tags` | タグの配列 | ✅ | `["Shape Display", "Haptics"]` |
| `summary` | 概要（日英対応） | ✅ | `{"ja": "概要文", "en": "Summary"}` |
| `heroImage` | メインビジュアル画像のパス | ❌ | `"/content/projects/.../images/HeroImage.png"` |
| `repoUrl` | リポジトリURL | ❌ | `"https://github.com/..."` |
| `demoUrl` | デモURL | ❌ | `"https://demo.example.com"` |
| `slidesUrl` | スライドURL | ❌ | `"https://slides.example.com"` |
| `relatedPapers` | 関連論文IDの配列 | ❌ | `["paper7", "paper12"]` |
| `language` | 表示言語（自動設定） | ✅ | 常に `"both"`（AIが自動設定） |
| `isPinned` | トップページに固定表示するか（自動設定） | ❌ | 最新プロジェクトは自動で `true`（AIが自動設定） |
| `featured` | ハイライト表示するか | ❌ | `true` または `false` |

#### 重要な注意点（AIが確認すべき点）

- **`slug` の変更は慎重に**: URLが変わるため、既存のブックマークやリンクが壊れます
- **`language` の設定**: プロジェクトは必ず日本語と英語の2つのバージョンを作成する運用のため、常に `"both"` を設定してください（ユーザーからの指定は不要）
- **`isPinned` の設定**: 最近のプロジェクトが自動でトップページに追加されるため、最新プロジェクトとして自動設定してください（ユーザーからの指定は不要）
- **`relatedPapers`**: 指定する論文IDは、論文データ（`content/papers/`）に存在する必要があります
- **日付形式**: `date` は必ず `YYYY-MM-DD` 形式（例: `"2025-03-01"`）で記載してください

#### 例：メタデータの更新

```json
{
  "id": "1",
  "title": {
    "ja": "FresnelShape",
    "en": "FresnelShape"
  },
  "slug": "fresnelshape",
  "status": "Published",
  "date": "2025-03-01",
  "tags": ["Shape Display", "Haptics"],
  "summary": {
    "ja": "指先への触覚提示により、様々な形状情報を提示できる小型軽量なデバイス",
    "en": "A small and lightweight device that can provide various shape information through tactile feedback to fingertips."
  },
  "heroImage": "/content/projects/project-1-fresnelshape/images/HeroImage.png",
  "relatedPapers": ["paper7", "paper12"],
  "language": "both",
  "isPinned": true,
  "featured": true
}
```

### 2. 本文の生成

#### ファイルの場所
```
content/projects/project-{id}-{slug}/content/
├── description_ja.md   # 日本語の本文（AIが生成）
└── description_en.md   # 英語の本文（AIが生成）
```

#### AIによる生成手順

1. **データソースから情報を抽出**
   - ユーザーが提供したPDF、ウェブサイトなどのデータソースから情報を抽出
   - 技術的な詳細を、わかりやすい言葉に置き換える準備を行う

2. **既存プロジェクトを参照**
   - `content/projects/project-1-fresnelshape/content/description_ja.md` や `description_en.md` などを参考にする
   - 既存プロジェクトと同じトーンや文体を意識する

3. **本文を生成**
   - 以下の4つのセクションで構成：
     - `## 背景` / `## Background`
     - `## アプローチ` / `## Approach`
     - `## 何がうれしいか` / `## Benefits`
     - `## 今後の展望` / `## Outlook`
   - 専門用語には適切な説明を追加（用語のすぐ後にかっこ書きで簡潔な説明文）
   - ですます調で記述し、わかりやすさを重視

4. **日英両方のファイルを生成**
   - `description_ja.md`（日本語版）
   - `description_en.md`（英語版）

#### AIによる本文生成時の注意事項

詳細は「[AIによる本文生成時の注意事項](#aiによる本文生成時の注意事項)」を参照してください。

#### Markdown記法の例

プロジェクト本文では以下のMarkdown記法が使用できます：

- **見出し**: `## 背景`, `## アプローチ` など
- **太字**: `**太字**`
- **リスト**: `- 項目1`, `- 項目2`
- **画像**: `![説明](/path/to/image.png)` または `![説明](/content/projects/.../images/image.gif)`
- **リンク**: `[リンクテキスト](https://example.com)`

### 3. 画像・アセットの追加・更新

#### ファイルの場所
```
content/projects/project-{id}-{slug}/
├── images/    # 画像ファイル（PNG、JPG、GIFなど）
└── assets/    # その他のアセットファイル
```

#### 更新手順

1. **画像を追加する場合**
   - `images/` ディレクトリに画像ファイルをコピー
   - ファイル名は既存の命名規則に合わせる（例: `HeroImage.png`, `background.gif`）

2. **`metadata.json` で画像パスを指定**
   - `heroImage` フィールドに画像パスを設定
   - 例: `"heroImage": "/content/projects/project-1-fresnelshape/images/HeroImage.png"`

3. **本文で画像を参照する場合**
   - `description_ja.md` または `description_en.md` 内で画像を参照
   - 例: `![説明](/content/projects/project-1-fresnelshape/images/background.gif)`

#### 画像ファイルの命名規則

- **メインビジュアル**: `HeroImage.png`（推奨）
- **背景画像**: `background.gif` など
- **アプローチ画像**: `approach.gif` など
- **その他**: 用途に応じて適切な名前を付ける

#### サポートされている画像形式

- PNG（`.png`）
- JPEG（`.jpg`, `.jpeg`）
- GIF（`.gif`）
- WebP（`.webp`）

#### 画像の推奨仕様

##### メインビジュアル画像（HeroImage）

**用途**: プロジェクトカードと詳細ページの上部に表示されるメイン画像

**推奨仕様**:
- **縦横比**: **3:2**（推奨）
  - プロジェクトカードで `aspect-[3/2]` として表示されます
  - 例: 1920px × 1280px, 1500px × 1000px など
- **解像度**: 
  - **最小**: 1200px × 800px
  - **推奨**: 1920px × 1280px
  - **最大**: 3840px × 2560px（4K）
- **ファイルサイズ**: **1MB未満**（推奨）
  - ページ読み込み速度を考慮し、可能な限り圧縮してください
  - オンラインツール（TinyPNG、Squooshなど）を使用して最適化することを推奨
- **形式**: PNG（透明度が必要な場合）または JPEG（写真など）
- **カラーモード**: RGB（Web表示用）

**注意点**:
- プロジェクト詳細ページでは `aspect-video`（16:9）として表示されますが、3:2の画像でも問題なく表示されます
- 画像が異なる縦横比の場合、中央部分がトリミングされます

##### 本文内の画像

**用途**: プロジェクト詳細ページの本文内に表示される画像（GIF、説明図など）

**推奨仕様**:
- **縦横比**: **16:9**（推奨）または用途に応じて自由
  - YouTube動画の埋め込みと統一感を出すため
  - GIFアニメーションなど、用途に応じて自由な縦横比でも問題ありません
- **解像度**:
  - **最小**: 800px × 450px（16:9の場合）
  - **推奨**: 1600px × 900px（16:9の場合）
  - **最大**: 2560px × 1440px（2K）
- **ファイルサイズ**: **2MB未満**（推奨）
  - GIFアニメーションは特にサイズが大きくなりやすいため、フレーム数や色数を調整してください
- **形式**: 
  - **静止画**: PNG（透明度が必要な場合）または JPEG
  - **アニメーション**: GIF（推奨）
- **カラーモード**: RGB（Web表示用）

##### 画像の最適化方法

1. **圧縮ツールの使用**
   - **PNG/JPEG**: [TinyPNG](https://tinypng.com/), [Squoosh](https://squoosh.app/)
   - **GIF**: [EZGIF](https://ezgif.com/optimize)

2. **解像度の調整**
   - 必要以上に高解像度の画像を使用しない
   - 表示サイズに合わせて適切な解像度にリサイズ

3. **形式の選択**
   - **写真**: JPEG（ファイルサイズが小さい）
   - **スクリーンショット・図**: PNG（画質が良い）
   - **アニメーション**: GIF

4. **ファイルサイズの目安**
   - メインビジュアル: 500KB〜1MB
   - 本文内画像: 1MB〜2MB
   - 可能な限り軽量化を心がける

##### 画像の品質チェック

以下の点を確認してください：

- ✅ 画像がぼやけていないか
- ✅ 必要以上に高解像度でないか（ファイルサイズとのバランス）
- ✅ ファイルサイズが適切か（目安: 1MB未満）
- ✅ 縦横比が適切か（3:2 推奨）
- ✅ カラーモードがRGBか（CMYKではない）

## よくある質問（FAQ）

### Q1. プロジェクトが表示されない

**A**: 以下の点を確認してください：

1. **`status` が `"Published"` になっているか確認**
   - `"Draft"` の場合は表示されません

2. **`language` の設定を確認**
   - プロジェクトは必ず日本語と英語の2つのバージョンを作成する運用のため、常に `"both"` が設定されているか確認

3. **ファイルパスが正しいか確認**
   - `metadata.json` が `info/` ディレクトリに存在するか
   - `description_ja.md` と `description_en.md` が `content/` ディレクトリに存在するか

4. **開発サーバーを再起動**
   - ファイルを追加した場合は、開発サーバーを再起動してください

---

### Q2. 画像が表示されない

**A**: 以下の点を確認してください：

1. **画像パスが正しいか確認**
   - `metadata.json` の `heroImage` フィールドのパスが正しいか
   - 本文内の画像パスが正しいか（`/content/projects/...` で始まる絶対パス推奨）

2. **画像ファイルが存在するか確認**
   - `images/` ディレクトリに画像ファイルが実際に存在するか

3. **ファイル名の大文字小文字を確認**
   - ファイル名は大文字小文字を区別します（例: `HeroImage.png` ≠ `heroimage.png`）

4. **画像形式がサポートされているか確認**
   - PNG、JPEG、GIF、WebP形式を確認

---

### Q3. トップページに表示されない

**A**: 以下の点を確認してください：

1. **`status` が `"Published"` になっているか確認**
   - 下書き状態では表示されません

2. **日付順で最新4件が表示される仕様**
   - トップページには最新4件のプロジェクトが自動で表示されます
   - より多くのプロジェクトを表示したい場合は、日付を新しいものに更新してください
   - **注意**: `isPinned` は自動設定されるため、ユーザーが手動で設定する必要はありません

---

### Q4. 関連論文が表示されない

**A**: 以下の点を確認してください：

1. **論文IDが正しいか確認**
   - `metadata.json` の `relatedPapers` フィールドに指定したIDが、論文データ（`content/papers/`）に存在するか確認

2. **論文の `language` 設定を確認**
   - 関連論文の `language` が `"both"` または現在の言語モードと一致しているか確認

3. **論文データの構造を確認**
   - 論文のメタデータファイル（`metadata.json`）が正しく配置されているか確認

---

### Q5. 言語切替がうまくいかない

**A**: 以下の点を確認してください：

1. **`language` フィールドの設定を確認**
   - プロジェクトは必ず日本語と英語の2つのバージョンを作成する運用のため、常に `"both"` が設定されていることを確認
   - AIが自動的に `"both"` を設定するため、ユーザーが手動で設定する必要はありません

2. **本文ファイルが存在するか確認**
   - `description_ja.md`（日本語用）
   - `description_en.md`（英語用）
   - 両方のファイルが存在することを確認

3. **ブラウザのキャッシュをクリア**
   - 開発中はブラウザのキャッシュが原因で古い内容が表示されることがあります

---

### Q6. タグでフィルタリングできない

**A**: 以下の点を確認してください：

1. **`tags` フィールドが配列形式になっているか確認**
   - 正しい形式: `["Tag1", "Tag2"]`
   - 間違った形式: `"Tag1, Tag2"`（文字列）

2. **タグ名にスペースや特殊文字が含まれていないか確認**
   - タグ名は英数字とハイフン、アンダースコアを推奨

3. **開発サーバーを再起動**
   - メタデータを変更した場合は、開発サーバーを再起動してください

---

### Q7. スラッグ（slug）を変更したい

**A**: スラッグの変更は慎重に行ってください：

1. **既存のリンクが壊れることに注意**
   - URLが変わるため、既存のブックマークや外部リンクが機能しなくなります

2. **変更手順**
   - `metadata.json` の `slug` フィールドを更新
   - 必要に応じてディレクトリ名も変更（`project-{id}-{new-slug}`）

3. **リダイレクトの設定を検討**
   - 既存のURLから新しいURLへのリダイレクトを設定することを検討してください

---

### Q8. 本文にMarkdownが反映されない

**A**: 以下の点を確認してください：

1. **Markdown記法が正しいか確認**
   - 見出し: `## 見出し`（半角の `#` を使用）
   - 画像: `![説明](/path/to/image.png)`（半角の `!` と `[]` を使用）

2. **ファイルのエンコーディングを確認**
   - UTF-8エンコーディングで保存されているか確認

3. **改行が正しく反映されているか確認**
   - Markdownでは、改行には空行が必要な場合があります

---

### Q9. ビルドエラーが発生する

**A**: 以下の点を確認してください：

1. **JSONファイルの構文エラーを確認**
   - `metadata.json` が正しいJSON形式になっているか確認
   - カンマの位置や引用符の閉じ忘れがないか確認

2. **ファイルパスのエラーを確認**
   - 画像パスやファイルパスが正しいか確認
   - 存在しないファイルを参照していないか確認

3. **エラーメッセージを確認**
   - ビルド時に表示されるエラーメッセージを確認し、該当箇所を修正

---

### Q10. 新規プロジェクトを追加したい

**A**: 以下の手順で新規プロジェクトを追加できます：

1. **ディレクトリを作成**
   ```
   content/projects/project-{id}-{slug}/
   ```

2. **必要なファイルを作成**
   - `info/metadata.json`
   - `content/description_ja.md`
   - `content/description_en.md`
   - `images/` ディレクトリ（必要に応じて）

3. **既存プロジェクトを参考にする**
   - 既存のプロジェクト（例: `project-1-fresnelshape`）をコピーして、内容を編集する方法が簡単です

4. **IDの重複を避ける**
   - `metadata.json` の `id` フィールドが既存プロジェクトと重複しないように注意

# メディアライブラリ運用ガイド

Decap CMS の画像アップロードは `public/images/uploads/` に保存されます。リポジトリをクローンした時点で必要なディレクトリが存在するよう、`.gitkeep` を配置しています。

## 1. ディレクトリ構成

```
public/
└─ images/
   └─ uploads/
       └─ .gitkeep
```

- CMS 内で画像をアップロードすると、生成されたファイルが `public/images/uploads/` に配置されます。
- ビルド後は `/images/uploads/<ファイル名>` で参照されます。

## 2. 運用ルール

- ファイル名は英数字とハイフンで統一し、スペースや日本語を避けてください。
- 一度アップロードした画像を差し替える場合は、CMS から新しいファイルをアップロードするだけでなく、不要になったファイルを手動で削除してリポジトリをクリーンに保つ運用を推奨します。
- 大きな画像はアップロード前に圧縮してください（目安 1MB 未満）。

## 3. Git 管理

- `uploads/` 配下のファイルは Git 管理対象です。CMS 経由で追加した画像は Pull Request に差分として含まれます。
- 差分レビュー時にファイルサイズに注意し、不要な大容量ファイルが混入していないか確認してください。

## 4. 代替テキスト

- CMS のフォームで `alt` テキストを必ず入力し、アクセシビリティを担保してください。
- 多言語対応が必要な場合は、各言語に対応するフィールドを設ける予定です（Issue 14 参照）。

以上を遵守することで、メディアライブラリと公開サイトの整合性が保たれます。

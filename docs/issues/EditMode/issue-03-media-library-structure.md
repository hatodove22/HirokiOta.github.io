# Issue: メディアライブラリ用ディレクトリの初期化

## 背景
画像アップロード先が不明確なままではコンテンツ運用が混乱する。Decap CMSのMedia Library設定とリポジトリ構造を合わせ、将来のS3連携などにも備えた土台を作る。

## スコープ
- `public/images/uploads/` などアップロード先ディレクトリを作成し、`.gitkeep` を配置
- `config.yml` の `media_folder` / `public_folder` を上記構成に合わせて更新
- READMEまたはDocsに、ローカルでの画像追加手順と運用ルールを追記

## 完了条件
- CMS上で画像をアップロードするとリポジトリ内の指定ディレクトリに配置される
- リポジトリを新規クローンした時に必要なディレクトリが存在しており、差分が出ない

- Issue 02のbackend設定を前提としつつ、メディアパスのみを扱う
- 将来的なCDN/S3連携は別Issueで検討する

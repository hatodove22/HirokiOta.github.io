# Issue: 共通編集UIコンポーネントとライブプレビュー土台の構築

## 背景
ニュース/プロジェクト/論文で共通する編集体験（サイドバー、一覧、詳細ペイン、プレビュー切替）をReactコンポーネントとして整備し、Decap CMSのプレビューAPIから再利用できるようにする必要がある。

## スコープ
- 左サイドバー、中央リスト、右詳細ペインのレイアウトとスタイルを実装
- プレビュー領域のPC/モバイル幅切替、ライト/ダークテーマ切替のUI骨格を提供
- Decap CMSの `CMS.registerPreviewTemplate` を利用し、プレースホルダー表示でもプレビューが描画されるところまで確認
- 共通フォームコンポーネント（公開設定ドロワー、保存ボタンなど）のUIスケルトンを用意

## 完了条件
- 各コレクションのプレビューで共通フレームワークが利用される
- UIコンポーネントがStorybookまたはドキュメント化され、以降のIssueでフィールドを追加しやすい状態になっている
- 現時点ではダミーデータでも良いが、ライブプレビュー切替の操作感が確認できる

## 依存/備考
- Issue #01でDecap CMS基盤が導入済みであること
- 詳細なフォーム項目やバリデーションは各コレクションのIssueで拡張する

# Issue: 多言語入力バリデーションとプレビュー切替の強化

## 背景
EditMode仕様では日本語/英語の切替や公開フラグに応じた必須入力制御が求められている。各コレクションで個別実装したロジックを共通化し、プレビューとの同期を強化する必要がある。

## スコープ
- `publish.ja` / `publish.en` の状態に応じて必須フィールドを動的に制御する共通バリデーションロジックを実装
- Decap CMSのカスタムウィジェットまたはプラグインを導入し、保存前に警告/エラーを表示
- プレビューのUIに言語トグルを追加し、現在の入力内容をリアルタイムに反映
- 画像タブの `alt_ja` / `alt_en` 入力必須を共通化し、未入力時の警告を表示

## 完了条件
- いずれかの言語のみ公開設定にした場合、不要なフィールドは任意扱いになり必要なフィールドが未入力だと保存できない
- プレビューで言語切替を操作すると対応する本文/メタ情報が即時に切り替わる
- バリデーション/プレビュー仕様がドキュメント化されている

## 依存/備考
- Issue #03〜#05で個別コレクションのフォームが完成していること
- 実装した共通ロジックは将来的なコレクション追加にも転用できるよう抽象化する

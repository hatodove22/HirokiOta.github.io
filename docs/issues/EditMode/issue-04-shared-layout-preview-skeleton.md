# Issue: 共通編集レイアウトとプレビュー連携の土台
> **ステータス:** 完了
> 運用変更（2025-09-29）: プレビュー連携先を Decap から `Portfolio_EditSystem_Proto` 内のプレビューに変更。以降のプレビュー要件は Proto 前提で定義。

## 背景
各コレクションのフォームを作り込む前に、編集画面の共通レイアウトとプレビュー切替の仕組みを薄く用意し、ニュースの最小ケースで動作を検証する。

## スコープ
- Decap CMSのカスタムプレビューで利用する共通レイアウトコンポーネントを作成（ヘッダー・プレビューエリア程度）
- プレビューとフォームの切替UIを最小構成で実装し、ニュースコレクションに適用
- Storybookなどのドキュメント化は範囲外とし、コンポーネント内コメントで利用方法を補足

## 完了条件
- ニュースコレクションで記事編集→プレビュー切替が可能で、Reactコンポーネントとして共通レイアウトが再利用できる
- プレビュー用のスタイルがベースのサイトデザインと大きく乖離していない

## 依存/備考
- Issue 01〜03で用意したDecap環境を前提とする
- 詳細なフォームUIやバリデーションは後続Issueで扱う

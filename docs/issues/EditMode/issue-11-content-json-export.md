# Issue: Markdown→JSON変換スクリプトのローカル実行

## 背景
既存のMarkdownコンテンツから静的サイト用JSONを生成するパイプラインはEditModeの重要な要素である。まずはローカルスクリプトとして確実に動作するものを用意し、CMSで編集した内容をサイトに反映できるようにする。

## スコープ
- Markdownファイルを読み込み、サイトで利用するJSONフォーマットに変換するスクリプトを実装
- ニュース・プロジェクト・論文の各ディレクトリを対象にし、最低限必要なフィールドを含める
- スクリプトの実行方法（例: `npm run export:json`）をpackage.jsonに追加し、Docsに手順を記載

## 完了条件
- ローカルでスクリプトを実行すると各コレクションのJSONが生成され、現行サイトのデータ構造と整合する
- 失敗時にはエラーログが出力され、原因調査が容易

## 依存/備考
- CMS側のフィールド実装が一通り揃っていることが望ましい（Issue 05〜10）
- ビルド統合やCI実行は後続Issueで扱う

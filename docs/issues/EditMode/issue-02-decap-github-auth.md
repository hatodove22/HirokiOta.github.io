# Issue: GitHub backend認証と手順ドキュメント整備
> **ステータス:** 完了

## 背景
管理画面を本番運用するにはGitHub backend認証が必須となる。開発者が同じ手順で認証を構築できるようにし、ローカル環境でも実際のGitHub Appを用いた動作確認が行える状態にする。

## スコープ
- `config.yml` をGitHub backend設定に更新し、必要な環境変数やURLを整理
- OAuthリダイレクト用の軽量なサーバーレス/Functionsエンドポイントの雛形を追加
- 認証周りのセットアップ手順を`docs`配下にまとめ、必要なSecrets一覧を明文化

## 完了条件
- 個人のGitHub Appを用いてDecap CMSにサインインできることをREADMEまたはDocsの手順通りに再現できる
- 認証に必要な環境変数が`.env.example`などで定義され、欠落時は明示的なエラーメッセージが表示される

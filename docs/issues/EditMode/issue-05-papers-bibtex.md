# Issue: 論文編集フォームとBibTeX同期フローの実装

## 背景
論文はBibTeXインポート・再インポート・エクスポートを伴う複雑な要件を持つ。Decap CMS側でのファイルアップロードと、GitHub Actionsによる自動変換処理を組み合わせる必要がある。

## スコープ
- 論文コレクションのフィールド定義（基本情報、著者、会議情報、リンク、タグ、関連プロジェクト、サムネイル、公開設定など）
- BibTeXアップロード用の`file`ウィジェットと再インポートガイダンスのUI
- プレビューで論文メタデータ、アブストラクト、多言語表示、関連プロジェクトを反映
- GitHub Actions `bib-import.yml` の設計草案とNodeスクリプトの雛形を配置（差分コミットまでの処理フロー）

## 完了条件
- 論文エントリがDecap CMS上で作成・編集可能であり、BibTeXファイルを添付できる
- プレビューがサイトの論文詳細レイアウトに沿って表示される
- GitHub Actionsの雛形が追加され、ローカルでNodeスクリプトを実行すると`.bib`→`content/papers/*.md`の変換が行える（テストデータで可）

- Actionsの本番運用はIssue #07でCI統合時に調整する

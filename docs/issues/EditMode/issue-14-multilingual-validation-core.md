# Issue: 多言語入力バリデーションの共通化

## 背景
EditModeでは日本語・英語など複数言語の入力を想定している。まずは保存時のバリデーションを共通化し、多言語フィールドの抜け漏れを防ぐ。

## スコープ
- ニュース・プロジェクト・論文のうち、多言語フィールドを持つ項目に共通バリデーションロジックを導入
- 言語ごとの必須条件（例: 日本語タイトルは必須、英語は任意など）を設定ファイルで管理できるようにする
- バリデーションエラーが発生した際のエラーメッセージを整備し、日本語/英語の両方で表示

## 完了条件
- 多言語フィールドが欠落している場合に保存がブロックされ、分かりやすいメッセージが表示される
- コレクションごとの必須条件が変更されても共通ロジックで対応できる

## 依存/備考
- 各コレクションのフィールド実装（Issue 05〜09）が完了していることが前提
- プレビュー側の言語切替は次Issueで扱う

## 方針更新（2025-09-29）
- バリデーションは Proto 側の共通ユーティリティとして実装（保存前ブロック/エラーメッセージ多言語対応）。
- ルール定義は `src/lib/types.ts`（必須/任意）と整合。Decap 側の保存フックは不要。

# 🧪 総合テスト: エディタ → プレビューの手動確認ガイド

## 📋 テスト概要
エディタにマークダウン文章を入力し、プレビューに適切に反映されているかを確認する総合テストです。

## 🚀 テスト手順

### 1. 開発サーバーの起動確認
```bash
# 開発サーバーが起動していることを確認
curl http://localhost:5174
# または ブラウザで http://localhost:5174 にアクセス
```

### 2. ブラウザでのアクセス
1. ブラウザで `http://localhost:5174` にアクセス
2. ページが正常に読み込まれることを確認

### 3. ニュース編集画面への遷移
1. 左側のサイドバーで「ニュース」をクリック
2. ニュース一覧画面が表示されることを確認
3. 「新規作成」ボタンをクリック
4. ニュース編集画面が表示されることを確認

### 4. エディタへのマークダウン入力
以下のマークダウン文章をエディタにコピー&ペーストしてください：

```markdown
# 総合テスト用マークダウン

## セクション1: 基本要素

これは**太字**と*斜体*のテストです。

### サブセクション1.1

- リストアイテム1
- リストアイテム2
- リストアイテム3

### サブセクション1.2

1. 番号付きリスト1
2. 番号付きリスト2
3. 番号付きリスト3

## セクション2: 高度な要素

> これは引用ブロックです。
> 複数行にわたる引用のテストです。

### コードブロック

```javascript
function testFunction() {
  console.log("Hello, World!");
  return "テスト完了";
}
```

### リンクと画像

[テストリンク](https://example.com)

## セクション3: 特殊文字

日本語のテスト: こんにちは、世界！

特殊文字: & < > " ' \`

### 長いテキストのテスト

これは非常に長いテキストのテストです。Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

## セクション4: ネストした構造

### 深いネスト

#### さらに深いネスト

##### 最も深いネスト

これは深いネストのテストです。

---

## テスト完了

以上で総合テスト用のマークダウン文章の入力が完了しました。
```

### 5. プレビュー確認
1. エディタの右側または下部にプレビューエリアがあることを確認
2. 以下の要素が正しく表示されているかを確認：

#### ✅ 確認項目

**基本要素:**
- [ ] H1見出し「総合テスト用マークダウン」が表示されている
- [ ] H2見出し「セクション1: 基本要素」が表示されている
- [ ] H3見出し「サブセクション1.1」が表示されている
- [ ] **太字**テキストが正しく表示されている
- [ ] *斜体*テキストが正しく表示されている

**リスト要素:**
- [ ] 箇条書きリスト（- で始まる項目）が正しく表示されている
- [ ] 番号付きリスト（1. で始まる項目）が正しく表示されている

**高度な要素:**
- [ ] 引用ブロック（> で始まる行）が正しく表示されている
- [ ] コードブロック（```で囲まれた部分）が正しく表示されている
- [ ] リンクが正しく表示されている

**特殊文字:**
- [ ] 日本語文字が正しく表示されている
- [ ] 特殊文字（& < > " ' \`）が正しく表示されている

**ネスト構造:**
- [ ] H4見出し「さらに深いネスト」が表示されている
- [ ] H5見出し「最も深いネスト」が表示されている
- [ ] 水平線（---）が正しく表示されている

### 6. データフロー確認
1. エディタでテキストを編集
2. プレビューがリアルタイムで更新されることを確認
3. マークダウン記法が正しくHTMLに変換されていることを確認

### 7. エラーハンドリング確認
1. 不正なマークダウン記法を入力（例：####### 不正な見出し）
2. エラーが適切に処理されることを確認
3. アプリケーションがクラッシュしないことを確認

## 🔧 トラブルシューティング

### エディタが見つからない場合
1. ブラウザの開発者ツールを開く（F12）
2. Consoleタブでエラーメッセージを確認
3. Networkタブでリソースの読み込み状況を確認

### プレビューが表示されない場合
1. エディタの内容が正しく入力されているか確認
2. プレビューボタンがある場合はクリック
3. ブラウザのキャッシュをクリア

### マークダウンが正しく変換されない場合
1. マークダウン記法が正しいか確認
2. 特殊文字のエスケープが必要か確認
3. エディタの設定を確認
